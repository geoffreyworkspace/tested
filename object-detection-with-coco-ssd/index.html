Object Detection with COCO-SSD

So, this is my starter code, we have two files, namely index.html & sketch.js. Moving on to the HTML file:

I have imported the ml5 library & the p5 library, I’m running the latest version as of writing this blog (p5.js – 1.2.0).

Moving on to the sketch.js file, we have a variable to store the image in, a preload function that just loads the image, and a setup function in which we’ve created a canvas with the create canvas function and an image function with the x coordinate & y coordinate.

sketch.js

Then we move on to create a variable to store our COCO-SSD model, let’s name this detector.

We then call the detector with the ml5’s objectDetector function and pass in the ‘cocossd’ as the model name.

To make things simple we’re using the preload function, without this we’d have to utilize callbacks or promises whichever you’d prefer.

Once we’ve completed this, we can now move on to create a detect function and pass it the image and a callback function that lets us know that the object has been detected, we’ll name it as gotDetections.

Let’s now write in the gotDetections function, here we create a function, and pass it two arguments, first the error & then the results as ml5 follow an error first callback mechanism.

We just handle the error by logging in into the console & move on.

To display the results, in this case, they are bounding boxes we’ll just write a for loop that’ll create the boxes by extracting their coordinates from the results array.

We can do this by drawing a rectangle and passing it the x coordinate, y coordinate, width, and height.

Also, we’ll add in the no fill as we don’t want to fill the rectangle, to display the ‘label’ of the object the model detected, we’ll pass in a text function with the x coordinate & the y coordinate to it.
